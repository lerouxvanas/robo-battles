# Patterns

Use this folder for small, reusable implementation patterns that are proven useful in this repo.

## Current patterns

- [react.md](./react.md) - default React component, state, effect, and UI flow guidance for this Vite app

## When to add a new pattern

- The same implementation shape appears in multiple places
- A new approach is expected to repeat soon
- The pattern is specific enough to guide real decisions, not generic advice

## Pattern doc format

Keep each pattern concise:
- what it is for
- when to use it
- when not to use it
- a small example or checklist

## Rule of thumb

Prefer an existing documented pattern before adding a new abstraction or helper layer.
